# 17. 外部連携設計書

## 連携方針

- MVPでは外部連携を最小にする
- 外部サービスのデータは信頼しない
- 連携失敗時にCRMの主要機能が止まらないようにする

## 連携候補

| 連携先 | 用途 | 優先度 |
|---|---|---|
| Google Drive | 素材バックアップ、資料参照 | Should |
| Google Calendar | 活動日程の管理 | Later |
| Slack | レビュー依頼・公開通知 | Should |
| Google Sheets | 初期データ移行 | Should |
| YouTube | ショート動画URL管理 | Later |
| Analytics | 公開ページ閲覧状況 | Later |

## Google Drive

### 用途

- 元素材の保管
- 共有済み資料の参照
- CRMへ登録する素材の移行元

### 注意

- Driveの権限とCRMの権限は一致しない
- Drive URLを公開サイトへ直接出さない
- 取り込んだ素材はCRM側で利用範囲を再確認する

## Slack

### 通知イベント

- レビュー依頼
- 差し戻し
- 承認
- 公開完了
- 公開ジョブ失敗

### メッセージ例

```text
レビュー依頼: 「第1回 問いの実践ゼミ」
担当: 佐藤
状態: REVIEW_REQUESTED
URL: /admin/reviews/xxx
```

## Google Sheets移行

### 用途

- 既存スプレッドシートの活動ログ取り込み
- 初期候補データの一括投入

### 取り込み仕様

- CSVインポート
- カラムマッピング
- 取り込み前プレビュー
- 重複チェック
- 取り込みログ保存

## Webhook

### 受信

MVPでは受信Webhookなし

### 送信

将来、公開完了時に外部通知Webhookを追加可能

## 失敗時動作

| 連携 | 失敗時 |
|---|---|
| Slack通知 | CRM内通知だけ残す |
| Drive取り込み | エラーを表示し再試行可能にする |
| Sheets取り込み | 部分成功を避け、トランザクションで全件ロールバック |
| Analytics | 公開機能には影響させない |
