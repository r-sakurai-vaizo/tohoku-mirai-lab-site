# 12. 権限設計書

## ロール

| ロール | 概要 |
|---|---|
| Admin | 全権限、ユーザー管理、設定変更 |
| Reviewer | 公開レビュー、承認、差し戻し |
| Editor | 活動・記録の作成編集、レビュー依頼 |
| Contributor | 下書き作成、素材登録 |
| Viewer | 管理画面閲覧のみ |

## 認可ルール

- API単位で認可を行う
- UI上でボタンを隠すだけでは不十分
- 自分のみ編集可能な対象は `owner_id = current_user.id` を確認する
- 公開、承認、権限変更は必ず監査ログへ保存する
- Disabledユーザーは全APIを利用不可

## 特別ルール

### 公開承認

- Editorは公開できない
- Reviewer以上が承認できる
- Adminは緊急公開できるが監査ログに理由を必須保存する

### 素材

- Contributorは素材登録できる
- 公開可否の変更はEditor以上
- `usage_scope = INTERNAL` の素材は公開コンテンツに紐付けても公開APIへ出さない

### サイト設定

- Adminのみ編集可能
- 更新時は差分を監査ログへ保存
