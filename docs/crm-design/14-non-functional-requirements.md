# 14. 非機能要件定義書

## 性能

| 項目 | 目標 |
|---|---|
| 管理画面初回表示 | 3秒以内 |
| 一覧検索 | 1秒以内 |
| 保存処理 | 2秒以内 |
| 公開サイトLCP | 2.5秒以内 |
| API p95 | 500ms以内 |

## 可用性

| 項目 | 目標 |
|---|---|
| 管理画面 | 月間99.5%以上 |
| 公開サイト | 月間99.9%以上 |
| DBバックアップ | 日次 |
| 復旧目標時間 | 4時間以内 |
| 復旧目標地点 | 24時間以内 |

## 拡張性

- 活動10,000件まで検索可能
- 記録10,000件まで検索可能
- 素材100,000件まで保存可能
- ユーザー100名まで対応
- 将来的に外部配信、分析、通知を追加できる

## 保守性

- API、DB、画面の仕様をドキュメント化する
- 状態遷移はenumで定義する
- 公開APIと管理APIを分離する
- 監査ログは共通関数で記録する
- 画面コンポーネントは一覧、フォーム、プレビューで再利用する

## ユーザビリティ

- 下書き保存は明示的に見える
- 公開前に必須チェックが見える
- エラーは原因と修正方法を表示する
- 長文編集は自動保存に対応する
- 一覧は検索と絞り込みを標準搭載する

## アクセシビリティ

- キーボード操作可能
- フォームにはlabelを付与
- エラーは色だけで伝えない
- コントラスト比4.5:1以上
- 画像にはaltを必須入力

## 互換性

- 最新版Chrome
- 最新版Edge
- 最新版Safari
- iPad幅で管理画面の主要操作が可能
- スマホでは閲覧と軽微な確認のみを想定

## データ品質

- slug重複不可
- 公開時は画像クレジット必須
- 公開時はsummary必須
- 公開時は大学公式誤認チェック必須
- 未公開素材は公開コンテンツに出せない
