# 01. 要件定義書

## 目的

東北大未来探求ラボの活動、記録、素材、公開承認を一元管理し、公開サイトへ安全に反映できるリッチCRMを構築する

## 背景

現在の公開サイトは静的JSONを読み込む構成で、活動や記録を増やすにはファイル編集が必要になる
今後、活動レポート、取材メモ、画像素材、公開承認、下書き管理が増えるため、運営メンバーがコードを触らず更新できる仕組みが必要になる

## 対象ユーザー

| ユーザー | 目的 |
|---|---|
| 共同代表 | 公開方針、承認、全体状況確認 |
| 編集担当 | 活動・記録の記事作成、素材整理 |
| 取材担当 | 取材メモ、関係者情報、活動登録 |
| レビュー担当 | 表現確認、公開可否判断 |
| 閲覧者 | 公開サイトで活動と記録を見る |

## スコープ

### 対象

- 活動管理
- 記録管理
- 素材管理
- 公開承認
- 公開サイト反映
- ユーザー・権限管理
- 監査ログ
- 検索・フィルタ
- 下書き保存

### 対象外

- 公開問い合わせフォーム
- 外部向け問い合わせ導線
- 決済
- 学生募集管理
- 大学公式システム連携
- 本格MA・営業CRM機能

## 機能要件

| ID | 要件 | 優先度 |
|---|---|---|
| FR-001 | 活動を登録、編集、削除できる | Must |
| FR-002 | 記録を登録、編集、削除できる | Must |
| FR-003 | 活動と記録を関連付けできる | Must |
| FR-004 | 画像・動画サムネイル・資料を素材として管理できる | Must |
| FR-005 | 下書き、レビュー中、承認済み、公開済みを管理できる | Must |
| FR-006 | 公開前レビューコメントを残せる | Should |
| FR-007 | 公開サイトには公開済みデータだけ出る | Must |
| FR-008 | 管理者がユーザー権限を変更できる | Must |
| FR-009 | 操作ログを記録できる | Must |
| FR-010 | 活動・記録を全文検索できる | Should |
| FR-011 | 公開予約ができる | Should |
| FR-012 | 画像のクレジットと利用範囲を管理できる | Must |

## 成功条件

- 運営メンバーがコード編集なしで活動・記録を下書き作成できる
- 承認されていないデータが公開サイトへ出ない
- 画像素材の出典・利用条件を公開前に確認できる
- 操作履歴から誰が何を変更したか追跡できる
- 公開サイトの表示は既存の静かなトーンを維持する

## 制約

- 公開サイトは営業感を出さない
- CRMは公開ナビに出さない
- 大学公式団体に見える表現を避ける
- 個人情報と未公開取材メモは公開データと分ける
